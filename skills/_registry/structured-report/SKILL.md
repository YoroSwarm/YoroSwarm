# Structured Report Skill

## 工作流程

1. 收集并整理输入数据（任务描述、上游任务产出、工作区文件等）
2. 根据报告类型选择合适的模板结构
3. 按照结构化格式组织内容
4. 使用 `create_workspace_file` 将报告保存为文件

## 报告模板

### 分析报告
```markdown
# [主题] 分析报告

## 执行摘要
[一段话总结关键发现]

## 背景
[任务背景和目标]

## 分析方法
[使用的分析方法和数据来源]

## 关键发现
### 发现 1: [标题]
- 详情：...
- 影响：...
- 建议：...

## 结论与建议
[总结和下一步建议]

## 附录
[支持数据和参考资料]
```

### 进度报告
```markdown
# [项目名] 进度报告

## 概述
- 报告周期：...
- 整体进度：X%

## 已完成工作
- ...

## 进行中工作
- ...

## 风险与阻塞
- ...

## 下一步计划
- ...
```

## 输出规范

1. **文件命名**：`report-[类型]-[日期].md`
2. **语言**：与输入数据的语言保持一致
3. **长度**：根据内容复杂度自适应，但确保结构完整
4. **引用**：所有数据和事实应标明来源
